# Стандарты документации

## Обзор

Этот навык обеспечивает создание детальной и понятной документации, которая позволяет разработчику реализовать функциональность без дополнительных вопросов.

## Инструкции

<instructions>
  * ВСЕГДА создавай детальные спецификации с примерами кода
  * ВСЕГДА описывай все параметры и возвращаемые значения
  * ВСЕГДА включай примеры использования для каждого публичного API
  * ВСЕГДА создавай диаграммы и визуализации для сложных концепций
  * ВСЕГДА используй четкое разделение на разделы с заголовками
  * НИКОГДА не оставляй документацию неполной
</instructions>

## Требования к документации

### Структура документации

```markdown
# Название функции/компонента

## Описание

Краткое описание того, что делает функция/компонент.

## Параметры

Детальное описание всех параметров с типами и примерами.

## Возвращаемое значение

Описание возвращаемого значения с типом.

## Примеры использования

Конкретные примеры кода с объяснениями.

## Подводные камни

Потенциальные проблемы и как их избежать.

## Связанные функции

Ссылки на связанные функции или компоненты.
```

### Примеры документации

#### Функция:

````typescript
/**
 * Вычисляет уровень пользователя на основе накопленного XP.
 *
 * @param xp - Количество опыта пользователя (должно быть >= 0)
 * @returns Объект с уровнем и прогрессом до следующего уровня
 *
 * @example
 * ```typescript
 * const result = calculateLevel(2500);
 * // { level: 15, currentXP: 2500, nextLevelXP: 3000, progress: 0.5 }
 * ```
 *
 * @remarks
 * Формула уровня: level = floor(sqrt(xp / 100))
 * Каждый уровень требует на 100 XP больше предыдущего
 */
function calculateLevel(xp: number): {
  level: number;
  currentXP: number;
  nextLevelXP: number;
  progress: number;
} {
  // реализация
}
````

#### Компонент:

````typescript
/**
 * Компонент карточки задачи для отображения в списке.
 *
 * @param task - Объект задачи с полями id, title, description, status
 * @param onComplete - Callback функция, вызываемая при завершении задачи
 * @param showActions - Показывать ли кнопки действий (по умолчанию: true)
 *
 * @example
 * ```tsx
 * <TaskCard
 *   task={{ id: 1, title: "Решить задачу", status: "active" }}
 *   onComplete={(id) => console.log(`Task ${id} completed`)}
 *   showActions={true}
 * />
 * ```
 */
interface TaskCardProps {
  task: Task;
  onComplete: (taskId: number) => void;
  showActions?: boolean;
}
````
