# Ясность UX (10/10)

## Обзор

Этот навык обеспечивает максимальную ясность при проектировании пользовательских интерфейсов, включая детальные пользовательские путешествия, обработку ошибок и упрощение сложных интерфейсов.

## Инструкции

<instructions>
  * ВСЕГДА создавай детальные пользовательские путешествия с конкретными действиями (шаг за шагом)
  * ВСЕГДА определяй все состояния ошибок с текстами сообщений
  * ВСЕГДА обрабатывай edge cases и граничные условия
  * ВСЕГДА описывай концепции фильтров и поиска (обязательно/необязательно/безразлично)
  * ВСЕГДА предлагай упрощение сложных интерфейсов (свертывание, пресеты, группировка)
  * НИКОГДА не оставляй UX решения на усмотрение разработчика
</instructions>

## Требования к UX ясности

### Пользовательские путешествия

Детальные пошаговые сценарии:

```
Пользовательское путешествие: Поиск и фильтрация задач

Шаг 1: Пользователь открывает страницу задач
  - Видит список задач (10 на странице)
  - Видит поле поиска вверху
  - Видит кнопку "Фильтры" справа от поиска

Шаг 2: Пользователь вводит текст в поле поиска
  - Вводит "математика" в поле поиска
  - Поиск выполняется автоматически после 500ms задержки (debounce)
  - Показывается индикатор загрузки

Шаг 3: Результаты обновляются
  - Список задач фильтруется по запросу "математика"
  - Показывается количество найденных результатов
  - Если результатов нет, показывается сообщение "Задачи не найдены"

Шаг 4: Пользователь открывает фильтры
  - Нажимает кнопку "Фильтры"
  - Открывается панель фильтров справа (drawer)
  - Видит фильтры: Предмет, Сложность, Статус, Дата

Шаг 5: Пользователь выбирает фильтры
  - Выбирает "Математика" в фильтре Предмет
  - Выбирает "Средняя" в фильтре Сложность
  - Нажимает "Применить"

Шаг 6: Результаты обновляются с учетом фильтров
  - Список задач обновляется
  - Показывается количество отфильтрованных результатов
  - Активные фильтры отображаются как теги над списком

Шаг 7: Пользователь сбрасывает фильтры
  - Нажимает "Сбросить" в панели фильтров
  - Все фильтры очищаются
  - Список возвращается к исходному состоянию
```

### Состояния ошибок

Все возможные ошибки с текстами:

```typescript
const ERROR_MESSAGES = {
  NETWORK_ERROR: {
    title: 'Ошибка подключения',
    message:
      'Не удалось подключиться к серверу. Проверьте интернет-соединение.',
    action: 'Повторить попытку',
  },
  NOT_FOUND: {
    title: 'Задачи не найдены',
    message:
      'По вашему запросу ничего не найдено. Попробуйте изменить параметры поиска.',
    action: 'Сбросить фильтры',
  },
  SERVER_ERROR: {
    title: 'Ошибка сервера',
    message: 'Произошла ошибка на сервере. Мы уже работаем над её устранением.',
    action: 'Обновить страницу',
  },
  VALIDATION_ERROR: {
    title: 'Ошибка валидации',
    message: 'Пожалуйста, проверьте правильность введенных данных.',
    action: 'Исправить',
  },
};
```

### Концепции фильтров

Трехпозиционный фильтр:

```typescript
type FilterState = 'required' | 'optional' | 'ignored';

interface FilterConfig {
  subject: FilterState; // required = только этот предмет, optional = может быть, ignored = любой
  difficulty: FilterState;
  status: FilterState;
}

// Примеры:
// { subject: 'required', difficulty: 'ignored', status: 'optional' }
// = Только математика, любая сложность, может быть выполнено или нет
```

### Упрощение сложных интерфейсов

Предложения по улучшению:

```
Проблема: Перегруженный фильтр с 15+ опциями

Решение 1: Свертывание
  - Показывать только 5 основных фильтров
  - Кнопка "Показать все" для остальных
  - Группировка по категориям

Решение 2: Пресеты
  - Быстрые фильтры: "Сегодня", "На этой неделе", "Просроченные"
  - Сохраненные фильтры пользователя

Решение 3: Группировка
  - Группа "Основные" (Предмет, Статус)
  - Группа "Дополнительные" (Дата, Теги, Автор)
```
